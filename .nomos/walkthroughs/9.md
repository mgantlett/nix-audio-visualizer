# Audio Calibration & Test Signal Mode

## Acceptance Criteria Coverage
- Created an `OscillatorNode` and `GainNode` synthetic test signal generator in `src/audio.ts` via the `toggleCalibrationMode` function.
- Implemented a 5-second logarithmic loop that sweeps from 20Hz to 15kHz, simulating real musical frequencies.
- Implemented a 120-BPM volume pulsing envelope on the `GainNode` to simulate percussive bass beats.
- Connected the `GainNode` to the `audioCtx.destination` to ensure the test signal plays audibly through the desktop speakers.
- Added a "Calibration Signal" toggle in the Music HUD Settings in `visualizer/index.html` which natively binds to the mode and refreshes the application on exit to cleanly restore microphone tracking.

## Verification
- Verified compilation via TypeScript.
- The UI triggers the calibration logic without error. The internal audio graph correctly unhooks from `getUserMedia` loopbacks and directly feeds the `OscillatorNode` to the visualizer logic, perfectly enabling precise VFD and EQ testing.
