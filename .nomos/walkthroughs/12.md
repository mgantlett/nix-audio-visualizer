# Enhance Calibration Diagnostic Modes

## Acceptance Criteria Coverage
- The `index.html` was updated to replace the single `Calibration Signal` checkbox with a `Diagnostic Mode` dropdown containing 4 states: `off`, `sweep`, `beat`, and `noise`.
- `state.ts` and `main.ts` were updated to handle the `state.testMode` string, successfully binding the new `<select>` HTML element.
- `src/audio.ts` was overhauled to route the 3 distinct modes in `toggleCalibrationMode`:
  1. `sweep`: Slowly sweeps from 20Hz to 22kHz over 15 seconds to allow full VFD column saturation.
  2. `beat`: Emits a 40Hz bass hit every 500ms to isolate Beat Detection algorithms.
  3. `noise`: Generates a pure Pink Noise buffer to saturate all frequency bands simultaneously.

## Verification
- Project successfully builds via TypeScript.
- Using the dropdown correctly spawns the specific audio nodes and routes them, properly tearing them down when switching modes or turning the system `off` (which restores the microphone loopback via a clean reload).
